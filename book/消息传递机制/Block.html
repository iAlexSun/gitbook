Block原理

什么是 Block

封装了一个函数实现的函数指针且将一些变量封装在block内部作为使用，调用block时相当于调用的是封装函数的地址。其实也就是封装函数的地址也就是函数指针。

Block 和 函数指针 的区别

block 在 ARC 和 MRC 下有什么不同

  • MRC

    • 非全局的block实现是在栈中生成的

    • 函数返回参数为 block 内部作为一个临时变量生成在栈上，返回时需要进行copy操作，

      否则调用其函数出现错误。

      • Block 需要手动释放

      • 使用__block修饰后捕获外部对象时在block内部实现没有进行强引用捕获的对象

  • ARC

    • 自动从栈中复制到堆上，ARC默认实现了block copy
    • 函数返回参数为 block 内部作为一个临时变量生成在栈上，自动进行了copy
    • 使用__block修饰后捕获外部对象时Block内部针对捕获的对象进行了强引用，容易出现循环引用的问题

  • 局部变量auto变量

如果捕获的的局部变量也就是auto变量，默认捕获的变量相当于只是将局部变量在内部生成一个相同的变量，在Block的构造函数中进行了复制操作，实际是值传递。所以在外部即使改变了变量，在block内部中捕获的变量也不会改变。

    };
    age = 20;
}

  • 静态变量 或 全局变量

    但是如果是 static 或者 全局变量，height 是指针传递，所以 static 或者 全局变量可以修改。因为静态变量和全局变量一直在内存中存在，可以使用指针针对内存地址进行修改。

{
    NSMutableArray *array = [NSMutableArray array];
    void (^block3)(void) = ^{
        [array addObject:@"1"];
    };
    block3();
}

是否可以通过编译？为什么？

可以通过编译且没有问题不需要添加__block，因为没有对 array 进行值的修改的操作。比如array = array1就是针对值得修改，但是 [array addObject:@"1"]; 没有对值进行修改，相当于对array 发送了一个 addObject 的objc_msgSend的消息，所以不需要添加 _block也可以执行。

__block如何修改值

__block 修饰了非对象变量后会包装成对象类型在 struct 中 __Block_byref_age_0 *age。age 变量已经被包装成了对象类型，经过__block修饰后传进 block 内部的 age 不是 auto 类型的变量则是被包装后的 __Block_byref_age_0 类型的 age 对象地址，所以 block 内部构造中不再是将 int age = 10 捕获进去，在 __main_block_impl_0 函数实现上也可以看到将 age 的地址传入 block ，那么在block 内部是通过 age->__forwarding->age = 20 去修改 age 的值，因为 __Block_byref_age_0 age 对象 中有一个属性是 __forwarding 指针指向自己，修改属性是 age 变量是通过 age->__forwarding->age 去修改属性的值。

为什么需要 __forwarding 来修改值？

因为有可能最开始的变量在栈上面后来复制到堆上，如果使用 age -> age 这样还是会去修改栈上的数值就可能出现问题了。但是如果使用 age -> __forwarding -> age 来修改的话就不会有这样的问题。因为 __forwarding 指针已经指向了堆上的 age 对象了，相当于栈上的 age 对象 __forwarding 指针指向堆的 age 对象，堆上的 age 对象的__forwarding指向了自己，所以采用 __forwarding 指针方式去修改属性的值不管什么情况都是修改堆上的值就不会出现只修改栈上的值的问题了。
