计时器实现原理

NSTimer 实现原理

NSTimer是基于CFRunLoopTimerRef实现的，将NSTimer注册到Runloop中后，RunLoop 会为其重复的时间点注册好事件。并不会在非常准确的时间点回调这个Timer。Timer 有个属性叫做 Tolerance (宽容度)，标示了当时间点到后，容许有多少最大误差。

为什么 NSTimer 在滑动时不好用

如果在DefaultMode的情况下timer是不会计时的，滑动后系统会调整为Track模式，正常情况下只要是中途切换了Mode的情况timer就会失效，需要在Timer添加到Common Mode类型下就可以了。因为Common Mode会包含DefaultMode和TrackMode两种模式。

CADisplayLink 使用及问题

基于 runloop 刷新率实现的定时器每秒60次也就是60FPS，一般可以用来检测页面的刷新率可以使用CADisplayLink，但是同样也会有循环引用的问题。

@property (nonatomic, strong) CADisplayLink *link;
- (void)viewDidLoad {
    [super viewDidLoad];
    self.link = [CADisplayLink displayLinkWithTarget:self selector:@selector(linkTest)];
    [self.link addToRunLoop:[NSRunLoop mainRunLoop] forMode:NSDefaultRunLoopMode];
}

- (void)linkTest {
    NSLog(@"%s",__func__);
}

- (void)dealloc {
    [self.link invalidate];
}

一般情况下日常如果使用 CADisplayLink 必须要添加在 runloop 中否则会失效，添加回调方法在 dealloc 中销毁 CADisplayLink 对象，看起来没问什么问题，但是在 CADisplayLink 初始化对象时和 NSTimer 一样会传入一个 target ，在内部强引用这个 target 如果是上面代码则就是强引用了这个VC，这样就会出现了循环引用问题self <-------> CADisplayLink 。

NSTimer 和 CADisplayLink 循环引用问题

//会产生循环引用
self.timer = [NSTimer scheduledTimerWithTimeInterval:1.0 target:self selector:@selector(timerTest) userInfo:nil repeats:YES];

NSTimer会造成循环引用，timer会强引用target即self，一般self又会持有timer作为属性，这样就造成了循环引用.如果timer只作为局部变量，不把timer作为属性呢？同样释放不了，因为在加入runloop的操作中，timer被强引用。而timer作为局部变量，是无法执行invalidate的，所以在timer被invalidate之前，self也就不会被释放。所以我们要注意，不仅仅是把timer当作实例变量的时候会造成循环引用，只要申请了timer，加入了runloop，并且target是self，虽然不是循环引用，但是self却没有释放的时机。如下方式申请的定时器，self已经无法释放了。

方案1

__weak typeof(self) weakSelf = self;
//不会产生循环引用，但是 Block 中需要使用弱引用对象 weakSelf 防止循环引用
self.timer = [NSTimer scheduledTimerWithTimeInterval:3 repeats:YES block:^(NSTimer * _Nonnull timer) {
     NSLog(@"%s",__func__);
      [weakSelf timerTest];
}];

使用 block 回调的形式可以避免 timer 强引用 self，但是只要是 block 就会捕获 self ,所以在 block 内部使用 weakSelf 弱引用 self 就可以解决这个为问题了。

方案2

可以使用NSProxy来解决这个问题，首先自定义一个 NSProxy 子类。

@property (nonatomic, weak) id target;
+ (instancetype)proxyWithTarget:(id)target {
    TestProxy *proxy = [TestProxy alloc];
    proxy.target = target;
    return proxy;
}

 self.timer = [NSTimer scheduledTimerWithTimeInterval:1.0 target:[TestProxy proxyWithTarget:self] selector:@selector(timerTest) userInfo:nil repeats:YES];

这样修改的话就可以了嘛？会发现出现了 crash 了因为 NSTimer 的 target 指向了 NSProxy 所以执行的方法就要找 NSProxy 是否实现这个方法如果没有则就会崩溃了。那么可以怎么解决这个问题呢？方案1可以在 NSProxy 内部实现 timerTest 方法做一些事情，但是这种方案会有问题，这个 NSProxy 之后如果再次使用需要添加不同的方法不是很好而且有的无法实现。方案2 NSProxy 的 API 默认支持消息转发机制相对于 NSObject 的好处在不需要查找流程直接进行消息转发即可如果没有则 crash，那么看一下如何实现 NSProxy 消息转发。

- (NSMethodSignature *)methodSignatureForSelector:(SEL)sel {
    return [self.target methodSignatureForSelector:sel];
}
- (void)forwardInvocation:(NSInvocation *)invocation {
    [invocation invokeWithTarget:self.target];
}

NSProxy 实现 forwardInvocation 和 methodSignatureForSelector 将消息转发给 target 就可以了，CADisplayLink 也可以使用 NSProxy 的这个方式解除循环引用问题，将消息转发给 VC。

self.link = [CADisplayLink displayLinkWithTarget:[TestProxy proxyWithTarget:self] selector:@selector(linkTest)];
[self.link addToRunLoop:[NSRunLoop mainRunLoop] forMode:NSDefaultRunLoopMode];

GCD Timer 如何使用及封装

这里可以参考 YYTimer 的一个自定义实现原理。

- (instancetype)initWithFireTime:(NSTimeInterval)start
                        interval:(NSTimeInterval)interval
                          target:(id)target
                        selector:(SEL)selector
                         repeats:(BOOL)repeats {
    self = [super init];
    _repeats = repeats;
    _timeInterval = interval;
    _valid = YES;
    _target = target;
    _selector = selector;

    __weak typeof(self) _self = self;
    _lock = dispatch_semaphore_create(1);
    _source = dispatch_source_create(DISPATCH_SOURCE_TYPE_TIMER, 0, 0, dispatch_get_main_queue());
    dispatch_source_set_timer(_source, dispatch_time(DISPATCH_TIME_NOW, (start * NSEC_PER_SEC)), (interval * NSEC_PER_SEC), 0);
    dispatch_source_set_event_handler(_source, ^{[_self fire];});
    dispatch_resume(_source);
    return self;
}

//取消计时器
- (void)invalidate {
    dispatch_semaphore_wait(_lock, DISPATCH_TIME_FOREVER);
    if (_valid) {
        dispatch_source_cancel(_source);
        _source = NULL;
        _target = nil;
        _valid = NO;
    }
    dispatch_semaphore_signal(_lock);
}
//开始
- (void)fire {
    if (!_valid) return;
    dispatch_semaphore_wait(_lock, DISPATCH_TIME_FOREVER);
    id target = _target;
    if (!target) {
        dispatch_semaphore_signal(_lock);
        [self invalidate];
    } else {
        dispatch_semaphore_signal(_lock);
        [target performSelector:_selector withObject:self];
        if (!_repeats) {
            [self invalidate];
        }
    }
}
